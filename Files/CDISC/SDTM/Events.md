# Events

Заплановані етапи протоколу, події та незаплановані інциденти, що відбуваються до або під час дослідження. Речі, що трапляються.

| Змінна | Опис | Додаткова інформація |
| --- | --- | --- |
| --TERM | Опис події дослівно зі слів пацієнта | |
| --MODIFY | Змінений термін, якщо AETERM незрозумілий | |

## AE

Зберігає інформацію про будь-які побічні ефекти, які трапилися із суб'єктом під час дослідження.

**<font color="red">One record per adverse event per subject</font>**

| Змінна | Опис | Додаткова інформація |
| --- | --- | --- |
| AEDECOD | Стандартизована назва події зі словника (Dictionary-Derived Term) | Використовується словник MedDRA |
| AEPRESP | Прапорець Y, якщо це питання було заздалегідь прописане в CRF | Наприклад, "Чи боліла у вас голова?" |
| AELOC | Локація події | |
| AESEV | Оцінка тяжкості | Шкала: MILD, MODERATE, SEVERE |
| AETOXGR | Оцінка токсичності | Шкала: MILD, MODERATE, SEVERE, LIFE-THREATENING, DEATH |

> Зазвичай використовують або AESEV, або AETOXGR. Використовувати дві змінні одночасно немає сенсу. Для AETOXGR існують спеціальні гайдлайни: <https://ctep.cancer.gov/protocoldevelopment/electronic_applications/ctc.htm>

| Змінна | Опис | Додаткова інформація |
| --- | --- | --- |

| AEACN / AEACNOTH | Дії, вжиті щодо препарату (Action Taken) або інші дії. | |
| AEREL | Причинно-наслідковий зв'язок з досліджуваним препаратом | |
| AERELNST | Зв'язок з ліками, що не досліджуються. | |
| AEOUT | Результат події — як все закінчилося | |
| AESTDY | Study day коли почався adverse event | |
| AEENDY | Study day коли закінчився adverse event | |
| AEDUR | Тривалість | Записується тільки якщо збиралася в CRF (не вираховується) |

### Словник MedDRA

Використовується для стандартизації скарг пацієнтів (AETERM → AEDECOD). Розкладає подію на рівні:

| Рівень | Опис | Додаткова інформація |
| --- | --- | --- |
| --LLT | Lowest Level Term | |
| --PT | Preferred Term | Зазвичай потрапляє в AEDECOD |
| --HLT | High Level Term | |
| --HLGT | High Level Group Term | |
| --SOC / --BODSYS | System Organ Class | Найвищий рівень (система органів), потрапляє в AESOC/AEBODSYS |

## DS

Зберігає будь які важливі події, які відбуваються на дослідженні

**<font color="red">One record per disposition status or protocol milestone per subject</font>**

Категорії подій, які зберігаються в DS:

* **DISPOSITION EVENT**
  * важливі події, що можуть відбутися, а можуть і не відбутися
* **PROTOCOL MILESTONE**
  * важливі події, що прописані за протоколом
  * якщо суб'єкт не пройшов цю подію, то його участь в дослідженні закінчується
* **OTHER EVENT**
  * важлива подія, що була не передбачена / небажана

| Змінна | Опис | Додаткова інформація |
| --- | --- | --- |
| DSCAT | Записується категорія важливої події | |

> Інформацію про COMPLETED статус важливої події зберігаємо, тільки якщо ця інформація збирається на CRF
