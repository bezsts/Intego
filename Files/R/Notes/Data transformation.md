## Pipe оператор

`%>%` (або `|>` у базовому R з версії 4.1) передає результат лівої частини як перший аргумент у праву. Дозволяє будувати зрозумілі ланцюжки операцій замість вкладених функцій.

```r
# Без pipe — читається зсередини назовні, незручно
filter(arrange(select(flights, year, month, day), dep_delay), dep_delay > 60)

# З pipe — читається зліва направо, як речення
flights %>%
  select(year, month, day, dep_delay) %>%
  arrange(dep_delay) %>%
  filter(dep_delay > 60)
```

## Огляд датафрейму

Перш ніж трансформувати дані, корисно швидко їх вивчити. Ось основні функції для цього.

```r
dplyr::glimpse(df)   # найкращий швидкий огляд: розмір, типи колонок, перші значення
str(df)              # структура — також з типами, але у форматі base R
summary(df)          # описова статистика по кожній колонці

head(df, n = 6)      # перші n рядків (за замовчуванням 6)
tail(df, n = 6)      # останні n рядків
head(df, n = -1)     # усі рядки, крім останнього

nrow(df)             # кількість рядків
ncol(df)             # кількість колонок
dim(df)              # розмір: c(рядки, колонки)
names(df)            # імена колонок (те саме, що colnames(df))
```

> **`glimpse()` vs `str()`:** `glimpse()` з dplyr зручніший — показує більше значень в один рядок і краще відображає типи tibble. Для швидкого огляду краще використовувати `glimpse()`.

## Рядки (Rows)

### `filter()` — фільтрація рядків

Залишає тільки рядки, що відповідають умові. Не змінює порядок і кількість колонок.

```r
# Рейси із затримкою вильоту більше 2 годин
flights %>%
  filter(dep_delay > 120)

# Кілька умов: рейси у січні АБО лютому
flights %>%
  filter(month == 1 | month == 2)

# Те саме, але через %in% — читається краще
flights %>%
  filter(month %in% c(1, 2))

# Рейси 1 січня з будь-якою затримкою
flights %>%
  filter(month == 1 & day == 1 & dep_delay > 0)
```

> **Увага з NA:** `filter()` автоматично виключає рядки, де умова дає `NA`. Якщо потрібно зберегти рядки з NA, додай: `filter(dep_delay > 120 | is.na(dep_delay))`.

**Корисні функції у filter():**

```r
# between() — перевірка належності до діапазону (включно з межами)
flights %>% filter(between(dep_delay, 60, 120))  # те саме що dep_delay >= 60 & dep_delay <= 120

# near() — безпечне порівняння дробових чисел (== часто дає хибний результат через float)
filter(near(x, y))           # перевіряє рівність з допуском
near(0.1 + 0.2, 0.3)        # TRUE  (0.1 + 0.2 == 0.3 дало б FALSE!)
near(1.5, 1.6, tol = 0.2)   # TRUE  — власний допуск через tol
```

> **Пріоритет операторів:** `&` виконується перед `|`, що може давати неочікувані результати. Завжди використовуй дужки для групування умов: `filter(AGE > 65 & (SEX == "F" | ARM == "Placebo"))`.

> **Порівняння двох колонок з NA:** якщо обидві колонки можуть містити NA, рядки де обидві NA не потраплять у результат через `==`. Щоб включити їх: `filter(col1 == col2 | (is.na(col1) & is.na(col2)))`.

### `arrange()` — сортування

Сортує рядки за значеннями колонок. За замовчуванням — за зростанням. `desc()` — за спаданням.

```r
# Сортування за зростанням (за кількома колонками — від лівої до правої)
flights %>%
  arrange(year, month, day, dep_time)

# Сортування за спаданням — рейси з найбільшою затримкою вгорі
flights %>%
  arrange(desc(dep_delay))
```

> NA завжди виявляється в кінці при будь-якому напрямку сортування.

### `distinct()` — унікальні значення

Прибирає дублікати — залишає тільки унікальні комбінації вказаних колонок.

```r
# Унікальні комбінації маршрутів (origin → dest)
flights %>%
  distinct(origin, dest)

# Унікальні маршрути + зберегти всі інші колонки першого рядка групи
flights %>%
  distinct(origin, dest, .keep_all = TRUE)

# Без аргументів — видаляє повністю дублікатні рядки
flights %>%
  distinct()
```

### `slice_*()` — вибір рядків за позицією

Сімейство функцій для вибору рядків за різними критеріями позиції. Найкраще поєднувати з `group_by()`.

| Функція | Що робить |
|---------|-----------|
| `slice_max(col, n = 1)` | рядки з найбільшим значенням |
| `slice_min(col, n = 1)` | рядки з найменшим значенням |
| `slice_head(n = 1)` | перші n рядків |
| `slice_tail(n = 1)` | останні n рядків |
| `slice_sample(n = 1)` | n випадкових рядків |

```r
# Рейс з найбільшою затримкою для кожного напрямку
flights %>%
  group_by(dest) %>%
  slice_max(arr_delay, n = 1) %>%
  select(dest, arr_delay)

# Рейс з найменшою затримкою для кожного напрямку
flights %>%
  group_by(dest) %>%
  slice_min(arr_delay, n = 1) %>%
  select(dest, arr_delay)

# Перший і останній рейс для кожного напрямку
flights %>% group_by(dest) %>% slice_head(n = 1)
flights %>% group_by(dest) %>% slice_tail(n = 1)

# Випадковий рейс для кожного напрямку
flights %>% group_by(dest) %>% slice_sample(n = 1)
```

## Колонки (Columns)

### `mutate()` — нові колонки

Додає нові колонки або змінює існуючі. За замовчуванням додає їх у кінець датафрейму.

```r
# Додати нові колонки в кінець
flights %>%
  mutate(
    gain  = dep_delay - arr_delay,   # наскільки рейс "надолужив" у повітрі
    speed = distance / air_time * 60 # швидкість у милях/год
  )

# Додати нові колонки перед першою колонкою
flights %>%
  mutate(
    gain  = dep_delay - arr_delay,
    speed = distance / air_time * 60,
    .before = 1
  )

# Залишити тільки колонки, що брали участь у розрахунку (.keep = "used")
flights %>%
  mutate(
    gain          = dep_delay - arr_delay,
    hours         = air_time / 60,
    gain_per_hour = gain / hours,
    .keep = "used"
  )
```

> **Параметри розміщення:**
> - `.before = 1` або `.before = col_name` — вставити перед колонкою
> - `.after = col_name` — вставити після колонки
> - `.keep = "used"` — залишити тільки задіяні колонки (зручно для перевірки)

### `select()` — вибір колонок

Вибирає (або виключає) колонки. Підтримує зручні helper-функції.

```r
# Вибрати конкретні колонки
flights %>% select(year, month, day)

# Вибрати діапазон колонок (від year до day включно)
flights %>% select(year:day)

# Виключити діапазон колонок
flights %>% select(!year:day)

# Вибрати тільки текстові колонки
flights %>% select(where(is.character))
```

**Helper-функції для вибору колонок:**

```r
# За початком назви
flights %>% select(starts_with("dep"))
flights %>% select(starts_with("dep") | starts_with("arr"))

# За кінцем назви
flights %>% select(ends_with("delay") | ends_with("time"))

# За наявністю рядка у назві
flights %>% select(contains("time"))

# Перейменування при виборі (нова_назва = стара_назва)
flights %>% select(tail_num = tailnum)

# Вибрати колонки з вектора назв
vars <- c("year", "month", "day")
flights %>% select(all_of(vars))

# Пронумерувати: вибрати колонки за патерном з числом
flights %>% select(num_range("race", 1:5))  # race1, race2, ..., race5
```

---

### `pull()` — витягти колонку як вектор

`select()` завжди повертає data frame. Якщо потрібен сам вектор — використовуй `pull()`.

```r
# Отримати унікальні значення ARM як вектор
flights %>%
  pull(carrier) %>%
  unique()

# pull() з іменуванням — повертає іменований вектор
flights %>%
  pull(carrier, name = flight)
```

> **`pull()` vs `$`:** `pull()` зручний у pipeline, `df$col` — для разового звернення поза pipeline.

---

### Вибір колонок через base R

У старому коді часто зустрічається base R синтаксис для роботи з колонками.

```r
# $ та [[]] — витягують колонку як вектор
df$AGE
df[["AGE"]]         # те саме, але через рядок — зручно, якщо ім'я в змінній

# [] — повертає data frame (навіть для однієї колонки)
df["AGE"]           # data frame з однією колонкою
df[c("AGE", "SEX")] # data frame з двома колонками

# Присвоєння через $
df$NEW_COL <- df$AGE + 1
```

> **Різниця `[[]]` і `[]`:** `df[["col"]]` — вектор, `df["col"]` — data frame. Для dplyr-pipeline краще використовувати `pull()` і `select()`.

### `rename()` — перейменування

Перейменовує колонки, зберігаючи всі інші. Синтаксис: `нова_назва = стара_назва`.

```r
# Перейменувати tailnum → tail_num, зберегти решту колонок
flights %>%
  rename(tail_num = tailnum)
```

> **`select()` vs `rename()`:** `select()` залишає тільки вказані колонки, `rename()` — зберігає всі.

---

### `relocate()` — переміщення колонок

Переміщує колонки в інше місце без їх видалення.

```r
# Перемістити time_hour і air_time на початок
flights %>%
  relocate(time_hour, air_time)

# Перемістити колонки, що починаються з "arr", перед dep_time
flights %>%
  relocate(starts_with("arr"), .before = dep_time)

# Перемістити після певної колонки
flights %>%
  relocate(time_hour, .after = day)
```

## Групи (Groups)

### `group_by()` — групування

Не змінює дані — тільки позначає датафрейм як згрупований. Усі наступні dplyr-функції будуть виконуватись окремо для кожної групи.

```r
# Згрупувати за місяцем (дані не змінюються, але в header з'являється "Groups: month [12]")
flights %>%
  group_by(month)

# Групування за кількома колонками
daily <- flights %>%
  group_by(year, month, day)
```

---

### `summarize()` — агрегація

Зводить кожну групу до одного рядка. Завжди використовується після `group_by()`.

```r
# Середня затримка і кількість рейсів по місяцях
flights %>%
  group_by(month) %>%
  summarize(
    avg_delay = mean(dep_delay, na.rm = TRUE), # na.rm = TRUE ігнорує NA
    n         = n()                             # n() — кількість рядків у групі
  )
```

**Корисні агрегатні функції:**

| Функція | Що рахує |
|---------|----------|
| `mean(x, na.rm = TRUE)` | середнє |
| `median(x, na.rm = TRUE)` | медіана |
| `sum(x, na.rm = TRUE)` | сума |
| `min(x)` / `max(x)` | мінімум / максимум |
| `n()` | кількість рядків |
| `n_distinct(x)` | кількість унікальних значень |
| `sd(x)` | стандартне відхилення |

---

### `ungroup()` — зняття групування

Знімає групування. Важливо після `group_by()` + `summarize()`, коли хочеш далі працювати з результатом без групування.

```r
daily <- flights %>%
  group_by(year, month, day)

daily %>%
  ungroup() %>%
  summarize(n = n()) # без групування поверне один рядок для всього датасету
```

---

### `.by` — альтернатива до `group_by()`

Починаючи з dplyr 1.1.0, можна вказати групування прямо в `summarize()` або `mutate()` через аргумент `.by`. Групування автоматично знімається після виконання функції.

```r
# Еквівалентно до group_by(month) %>% summarize(...)
flights %>%
  summarize(
    delay = mean(dep_delay, na.rm = TRUE),
    n     = n(),
    .by   = month
  )

# Групування за кількома колонками
flights %>%
  summarize(
    delay = mean(dep_delay, na.rm = TRUE),
    n     = n(),
    .by   = c(origin, dest)
  )
```

> **`group_by()` vs `.by`:** `group_by()` зручніше, коли потрібно зробити кілька операцій з тією самою групою. `.by` краще для разового групування — не треба думати про `ungroup()`.
